# plan-manager Skill

一个基于 Markdown 的工作计划管理工具。计划文件存储在本 skill 目录内部，方便整体迁移。

## 触发条件

当用户提到以下意图时使用此 skill：
- 创建/新建工作计划
- 查看/列出计划
- 添加执行步骤
- 标记步骤完成
- 添加操作日志/备注
- 归档已完成计划
- 任何涉及"计划"、"todo"、"待办"、"工作安排"的管理操作

## 文件存储位置

- **进行中的计划**: `.claude/skills/plan-manager/todo/`
- **已完成归档**: `.claude/skills/plan-manager/done/`

## 计划文件格式

每个计划是一个独立的 Markdown 文件，格式如下：

```markdown
# {计划标题}

## 计划信息
**计划编号**: PLAN-YYYYMMDD-XXX
**创建日期**: YYYY-MM-DD
**状态**: {待开始|进行中|已完成}
**完成时间**: YYYY-MM-DD（目标日期，可选）

## 计划目标
{具体目标描述}

## 执行步骤
1. [ ] 步骤描述
2. [x] 已完成的步骤描述

## 操作日志
- YYYY-MM-DD HH:MM:SS - 最新的操作记录
- YYYY-MM-DD HH:MM:SS - 较早的操作记录
```

## 命名规则（重要约束）

- **文件名 = 计划标题**，如 `吃饭.md`、`项目开发计划.md`
- **禁止**使用 PLAN-ID（形如 `PLAN-20260518-001`）作为文件名
- PLAN-ID 仅写入文件内部的「计划信息」部分作为唯一标识

## 操作规范

### 1. 创建计划

**步骤：**
1. 读取 `todo/` 目录下已有文件，确定今天的序号以生成唯一 PLAN-ID（格式 `PLAN-YYYYMMDD-XXX`，XXX 从 001 递增）
2. 在 `todo/` 目录创建 `{标题}.md` 文件
3. 按上述格式填写内容，状态设为「待开始」
4. 操作日志写入创建记录

**示例调用：** "创建一个计划：回收所有旧电脑，目标完成日期 2026-06-01"

### 2. 添加执行步骤

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 在「执行步骤」部分末尾追加新步骤（或按指定位置插入）
3. 重新编号所有步骤
4. 如果状态为「待开始」，更新为「进行中」
5. 在操作日志中记录本次添加

**示例调用：** "给'回收所有旧电脑'计划添加步骤：联系IT部门收集电脑信息"

### 3. 标记步骤完成

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 将指定步骤的 `[ ]` 改为 `[x]`
3. 检查是否所有步骤都已完成：若是，将状态更新为「已完成」
4. 在操作日志中记录本次完成

**示例调用：** "标记'回收所有旧电脑'的步骤1为已完成"

### 4. 添加操作日志

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 在「操作日志」部分的**最前面**（紧跟 `## 操作日志` 标题之后）插入新日志条目
3. 格式：`- YYYY-MM-DD HH:MM:SS - {日志内容}`

**示例调用：** "给'回收所有旧电脑'添加日志：第一步已经完成，5个部门都很配合"

### 5. 列出计划

**步骤：**
1. 读取 `todo/` 目录下所有 `.md` 文件（列出未完成计划）；或读取 `done/` 目录（列出已完成计划）
2. 从每个文件中提取：**计划编号（PLAN-ID）**、标题、状态、创建日期、目标完成时间
3. 按目标完成时间排序，展示给用户
4. **展示时必须包含 PLAN-ID**，方便用户通过编号引用计划

**输出格式示例：**
| # | 计划编号 | 计划 | 预计完成 | 工作内容 |
|---|----------|------|----------|----------|
| 1 | PLAN-20260326-001 | 电脑升级后设备回收计划 | 2026-06月 | 成都深圳31台美术升级退换机器，按批次回收转给运营组使用 |
| 2 | PLAN-20260330-001 | 云桌面扩容跟进计划 | 2026-05月下旬 | 基地同学云桌面存储扩容，先扩云固态再迁移XL2 5080 |
| 3 | PLAN-20260506-001 | AI问答机器人对外开放调研 | - | 调研AI问答机器人对外开放的技术可行性、安全合规及开放范围 |

**展示要求：**
- **必须展示 PLAN-ID**（计划编号），方便用户通过编号引用计划
- **必须展示预计完成时间**，若未设置则显示"-"
- **工作内容简要展示**：从「计划目标」中提炼一句话概述，不超过30字为佳
- 按创建日期排序

**示例调用：** "列出所有进行中的计划"、"展示已完成的计划"、"当前有哪些未完成工作计划"

### 6. 查看计划内容

**步骤：**
1. 读取 `todo/{计划标题}.md` 的完整内容
2. 展示给用户

**示例调用：** "查看'回收所有旧电脑'计划的详细内容"

### 7. 归档计划

**步骤：**
1. 读取 `todo/{计划标题}.md`，确认状态为「已完成」
2. 如果未完成，提示用户不能归档
3. 将文件从 `todo/` 移动到 `done/` 目录
4. 在移动后的文件中追加归档日志记录

**示例调用：** "把'回收所有旧电脑'计划归档"

## 状态流转规则

```
待开始 ──(添加步骤/开始工作)──> 进行中 ──(所有步骤完成)──> 已完成
```

- 创建时默认「待开始」
- 首次添加步骤或用户明确开始时，自动转为「进行中」
- 所有步骤标记为 `[x]` 时，自动转为「已完成」
- 归档操作只允许对「已完成」状态的计划执行

## 输出格式

执行操作后，向用户简洁报告结果，例如：
- "已创建计划'回收所有旧电脑'（PLAN-20260518-001），目标完成日期：2026-06-01"
- "已为'回收所有旧电脑'添加步骤3：执行回收操作"
- "步骤1已标记完成。当前进度：1/3"
- "已归档计划'回收所有旧电脑'到 done/ 目录"

## Git 自动同步

本 skill 目录（`.claude/skills/plan-manager/`）是一个独立的 git 仓库，用于将工作计划同步到远端，方便在其他平台实时查看。

### 自动同步规则

**写操作前先拉取远端更新（防止冲突）：**
1. `git pull`（拉取远端最新变更，确保本地是最新状态）

**写操作完成后自动推送：**
2. `git add -A`（添加所有变更）
3. `git commit -m "{操作描述}"`（提交，commit message 简要描述本次操作）
4. `git push`（推送到远端）

**注意：仅在需要修改计划文件时才执行 pull，纯读取/查看操作不触发 pull，减少同步频率。**

**触发同步的操作包括：**
- 创建计划
- 添加/修改执行步骤
- 标记步骤完成
- 添加操作日志
- 归档计划（移至 done/）
- 更新计划信息（目标、预计完成时间等）

**commit message 格式：**
- 创建计划：`新建计划: {标题}`
- 更新计划：`更新计划: {标题} - {操作概述}`
- 完成计划：`完成计划: {标题}`
- 归档计划：`归档计划: {标题}`
- 批量操作：`批量更新: {概述}`

### 初始化

如果 `.claude/skills/plan-manager/` 目录尚未初始化为 git 仓库，需先执行：
```bash
cd .claude/skills/plan-manager/
git init
git add -A
git commit -m "初始化 plan-manager"
git remote add origin {远端地址}
git push -u origin main
```

### 注意事项
- 如果 push 失败（如网络问题），不影响本地操作，下次操作时会一并推送
- 不要在 commit message 中包含敏感信息
- `.gitignore` 中不需要忽略任何文件（所有计划文件都应同步）
