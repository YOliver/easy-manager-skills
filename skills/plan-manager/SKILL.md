# plan-manager Skill

一个基于 Markdown 的工作计划管理工具，计划文件存储在 skill 目录内部，方便整体迁移，支持 Git 自动同步到远端，方便在其他平台实时查看工作计划状况。

## 触发条件

当用户提到以下意图时使用此 skill：
- 创建/新建工作计划
- 查看/列出计划
- 添加执行步骤
- 标记步骤完成
- 添加操作日志/备注
- 归档已完成计划
- 任何涉及"计划"、"todo"、"待办"、"工作安排"的管理操作

## 功能特性

- **计划管理**: 创建、查看、管理工作计划（.md 格式）
- **唯一编号**: 自动生成 `PLAN-YYYYMMDD-XXX` 格式的唯一计划编号
- **步骤跟踪**: 为每个计划添加执行步骤，支持完成状态跟踪
- **操作日志**: 自动记录所有操作历史（带时间戳）
- **状态管理**: 自动状态流转（待开始 → 进行中 → 已完成）
- **自动归档**: 计划完成后自动移至 done 目录，无需手动操作
- **Git 同步**: 每次操作后自动 commit + push，远端实时同步

## 文件存储位置

- **进行中的计划**: 本skill目录/todo/
- **已完成归档**: 本skill目录/done/

## 计划文件格式

每个计划是一个独立的 Markdown 文件，具体格式参见 `TEMPLATE.md`。

创建新计划时，读取 `TEMPLATE.md` 获取模板结构，按模板填写内容。

## 命名规则

- **文件名 = 计划标题**，如 `项目开发计划.md`
- **禁止**使用 PLAN-ID（形如 `PLAN-20260518-001`）作为文件名
- PLAN-ID 仅写入文件内部的「计划信息」部分作为唯一标识

## 操作规范

### 1. 创建计划

**步骤：**
1. 读取 `todo/` 目录下已有文件，确定今天的序号以生成唯一 PLAN-ID（格式 `PLAN-YYYYMMDD-XXX`，XXX 从 001 递增）
2. 在 `todo/` 目录创建 `{标题}.md` 文件
3. 按上述格式填写内容，状态设为「待开始」
4. 操作日志写入创建记录

### 2. 添加执行步骤

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 在「执行步骤」部分末尾追加新步骤（或按指定位置插入）
3. 重新编号所有步骤
4. 如果状态为「待开始」，更新为「进行中」
5. 在操作日志中记录本次添加

### 3. 标记步骤完成

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 将指定步骤的 `[ ]` 改为 `[x]`
3. 检查是否所有步骤都已完成：若是，将状态更新为「已完成」，并立即执行自动归档
4. 在操作日志中记录本次完成

### 4. 添加操作日志

**步骤：**
1. 读取 `todo/{计划标题}.md`
2. 在「操作日志」部分的**最前面**（紧跟 `## 操作日志` 标题之后）插入新日志条目
3. 格式：`- YYYY-MM-DD HH:MM:SS - {日志内容}`
4. **强制排序规则**：添加日志后，必须确保「操作日志」部分按**时间倒序**排列（最新的在最前面）。排序优先级：**年 > 月 > 日 > 时 > 分 > 秒**。如果日志顺序错误，必须立即重新排序后再保存文件。

### 5. 列出计划

**步骤：**
1. 读取 `todo/` 目录下所有 `.md` 文件（列出未完成计划）；或读取 `done/` 目录（列出已完成计划）
2. 从每个文件中提取：**计划编号（PLAN-ID）**、标题、状态、创建日期、目标完成时间
3. 按目标完成时间排序，展示给用户
4. **展示时必须包含 PLAN-ID**，方便用户通过编号引用计划

**输出格式：**

| # | 计划编号 | 计划 | 预计完成 | 工作内容 |
|---|----------|------|----------|----------|
| 1 | PLAN-20260326-001 | 电脑升级后设备回收计划 | 2026-06月 | ... |

**展示要求：**
- **必须展示 PLAN-ID**（计划编号）
- **必须展示预计完成时间**，若未设置则显示"-"
- 按目标完成时间排序（紧急的排前面）

### 6. 查看计划内容

**步骤：**
1. 读取 `todo/{计划标题}.md` 或 `done/{计划标题}.md` 的完整内容
2. 展示给用户

### 7. 自动归档

计划所有步骤标记完成后，状态自动变为「已完成」。系统**自动执行**以下归档操作：
1. 将文件从 `todo/` 移动到 `done/` 目录
2. 在移动后的文件中追加归档日志记录
3. 用户无需手动触发归档

## 状态流转规则

```
待开始 ──(添加步骤/开始工作)──> 进行中 ──(所有步骤完成)──> 已完成 ──(自动归档)──> done/
```

- 创建时默认「待开始」
- 首次添加步骤或用户明确开始时，自动转为「进行中」
- 所有步骤标记为 `[x]` 时，自动转为「已完成」
- 「已完成」状态立即触发自动归档（移至 `done/` 目录）

## 输出格式

执行操作后，向用户简洁报告结果，例如：
- "已创建计划'项目开发计划'（PLAN-20260518-001），目标完成日期：2026-06-01"
- "已为'项目开发计划'添加步骤3：执行回收操作"
- "步骤1已标记完成。当前进度：1/3"
- "所有步骤已完成，计划'项目开发计划'已自动归档到 done/ 目录"

## Git 自动同步

本 skill 目录是一个独立的 git 仓库，用于将工作计划同步到远端，方便在其他平台实时查看。

### 自动同步规则

**写操作前先拉取远端更新（防止冲突）：**
1. `git pull`（拉取远端最新变更）

**写操作完成后自动推送：**
2. `git add -A`
3. `git commit -m "{操作描述}"`
4. `git push`

**触发同步的操作包括：**
- 创建计划
- 添加/修改执行步骤
- 标记步骤完成
- 添加操作日志
- 自动归档（移至 done/）
- 更新计划信息（目标、预计完成时间等）

**commit message 格式：**
- 创建计划：`新建计划: {标题}`
- 更新计划：`更新计划: {标题} - {操作概述}`
- 完成计划：`完成计划: {标题}`
- 归档计划：`归档计划: {标题}`
- 批量操作：`批量更新: {概述}`

**注意：** 仅在需要修改计划文件时才执行 pull，纯读取/查看操作不触发 pull。

### 初始化

如果本 skill 目录尚未初始化为 git 仓库，需在本 skill 目录下执行：
```bash
git init
git add -A
git commit -m "初始化 plan-manager"
git remote add origin {远端地址}
git push -u origin main
```

### 注意事项
- 如果 push 失败（如网络问题），不影响本地操作，下次操作时会一并推送
- 不要在 commit message 中包含敏感信息

## API 参考

### create_plan(title, target_date="")
创建新的工作计划。

**参数:**
- `title`: 计划标题（用作文件名）
- `target_date`: 目标完成日期（可选）

**返回:** 操作结果，包含 plan_id

### add_step(plan_title, step_description, position=None)
为指定计划添加执行步骤。

**参数:**
- `plan_title`: 计划标题
- `step_description`: 步骤描述
- `position`: 插入位置（可选，默认追加末尾）

### complete_step(plan_title, step_number)
标记指定计划的步骤为已完成。

**参数:**
- `plan_title`: 计划标题
- `step_number`: 步骤编号

### add_log(plan_title, log_message)
为指定计划添加操作日志。

**参数:**
- `plan_title`: 计划标题
- `log_message`: 日志消息

### list_plans()
列出所有未完成工作计划。

**返回:** 计划列表（含编号、标题、预计完成时间、工作内容概述）

### show_plan_content(plan_title)
展示指定工作计划的完整文档内容。

**参数:**
- `plan_title`: 计划标题

### move_to_done(plan_title)
将已完成的工作计划归档到 done 目录（由自动归档调用，一般无需手动执行）。

**参数:**
- `plan_title`: 计划标题

## 存储结构

```
plan-manager/
├── SKILL.md          ← 技能描述与行为指令（本文件）
├── TEMPLATE.md       ← 计划文件格式模板
├── todo/             ← 进行中的计划
│   ├── 计划A.md
│   └── 计划B.md
└── done/             ← 已完成归档
    └── 已完成计划.md
```
